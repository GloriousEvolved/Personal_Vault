### HTML
1. > [!faq]- Что такое *HTML* и для чего он используется?
    >**HTML** - (HyperText Markup Language) язык гипертекстовой разметки. Каркас любой веб-страницы или веб-приложения. С помощью HTML добавляется весь контент на веб-странице - текст, мультимедиа (фото, видео, аудио), кнопки, поля ввода и гиперссылки. Без HTML страница не может существовать. 

  2. > [!faq]- Как *HTML* связан с *CSS* и *JavaScript*?
      > **HTML**, **CSS** и **JavaScript** - это 3 основных языка, с помощью которых создаётся веб-страница. HTML - это содержимое страницы, CSS - отвечает за внешний вид страницы и подключается внутри HTML файла с помощью тега `<link rel='stylesheet' href='путь/до/стилей.css'>`, а JavaScript - отвечает за реакции на действия пользователя и подключается через тег `<script>`.

 3. > [!faq]- Из чего состоит *HTML*-документ?
     >HTML-документ — это каркас, состоящий из трёх главных частей:  
**объявления (<DOCTYPE html>)**, **головы/настройки (head)** и **тела/содержания (body)**.
     >```HTML
     ><DOCTYPE html> // Сначала идёт <DOCTYPE html> - объявление типа документа. Этот тог сообщает браузеру на каком языке написан документ.
     ><html lang="ru"> // Дальше идёт тег <html>. Это корневой тег для всего что отображается на веб-странице.
     >	<head> // Тег <head> - это раздел с метаданными. Здесь указываются кодировка, название вкладки, подключение стилей, подключение иконки, шрифтов и скриптов.
     >		<meta charset="UTF-8">
     >		<title>Название страницы</title>
     >	</head>
     >	<body> // Тег содержание страницы. Здесь размещаются заголовки, текст, мультимеди, кнопки, ссылки, формы, таблицы и т.д.
     >		<h1>Привет, мир!</h1>
     >		<p>Это мой первый HTML-документ.</p>
     >	 </body>
     >	</html>
     >```     
 
4. > [!faq]- Что означает «дерево элементов» (*DOM*-дерево)?
    >  «Дерево элементов» — это структура, которую браузер строит, когда читает/[[#^16a100|парсит]] HMTL-код, чтобы в дальнейшем работать с ней. С помощью DOM наглядно видно вложенность тегов и благодаря DOM JavaScript может манипулировать структурой (добавлять, удалять ветви (node) или менять их содержимое. 
    > Оно показывает, какие элементы есть на странице и их вложенность.
    > Браузер не просто видит текст с тегами вроде:
    > ```html
    > <body>
    > <h1>Привет</h1>
    > <p>Это пример</p>
    > </body>
    > ```
    > Он строит иерархию\структуру объектов, где каждый тег - это узел/ветвь дерева(node). Визуально выглядит так (ещё один наглядный [пример](https://learn.javascript.ru/dom-nodes)):
    > ```javascript
    > html
    > └── body
    >       ├── h1
    >       │    └── "Привет"
    >       └── p
    > 		    └── "Это пример"
    > ```
    > Эта структура названа деревом, потому что имеет общий корень в виде тега `<html>` от которого, словно ветви идут `body` и `head`, а от них ещё теги. Тем самым образую структуру, похожую на дерево.
    > 
    > Браузер превращает HTML в **DOM (Document Object Model)** - специальный объект, с которым может работать **JavaScript**.
    > Например:
    > ```javascript
    > document.body.style.background = "lightblue";
    > ```
    > В это примере `document` — это всё дерево страницы, а `body` — один из его узлов, к которому мы обращаемся напрямую.

3. > [!faq]- Что происходит при парсинге *HMTL* документа браузером?
    > Парсинг HTML - это процесс, при котором бразуер читает HTML код и превращает его в [[#^e3863d|DOM]], с которой потом может работать и он сам, и JavaScript.
    > При парсинге HTML документа, браузер читает код, разбирает его по смыслу, строит модель страницы (DOM) и подготавливает всё, чтобы нарисовать страницу на экране.
    >^16a100

4. > [!faq]- Какие основные секции входят в HTML-документ?
    > В HTML входят секции `<html>` - корневой контейнер, `<head>` - настройки, `<body>` - содержание.
