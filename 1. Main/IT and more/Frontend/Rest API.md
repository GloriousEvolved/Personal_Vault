## Клиент серверная ахитектура/модель

	Сервер - это источник данных(какое-то железо на котором есть наше серверное приложение).
	
<br/>

	Клиент - компьютер, сервер, браузер, телефон, программа.

^c73853

Клиент инициирует запрос, а сервер его принимает, обрабатывает и отправляет ответ с какими-то данными.(смотри схему). Эти данные в Rest называются *ресурсы* или же *endpoint*. Ресурсами может быть что угодно: текст, аудио, картинка или видео.
> [!example] Например: это может быть HTML документ или JSON файл с данными. 

Ресурсы имеют уникальные идентификатор, которые обычно представлен в виде URL. Пример: `https://example.com/api/v3/products` или `https://example.com/api/v3/users`^eb89c0

Над ними мы можем выполнять CRUD операции. 

	CRUD(Create, Read, Update, Delete) - термин для 4 операций: создание, чтение, обновление и удаление элемента.

Сервер взаимодействует с базами данных(БД), определяет протоколы взаимодействия и определяет API. Я с ним взаимодействую с помощью каких-то определённых действий. Я могу общаться с сервером в формате "Вопрос - Ответ". 
> [!example] Пример действий с сервером.
> Например: 1) попросить у сервера данные, 2) выполнить какое-то действие (зарегестрировать нового пользователя, провести оплату, добавить товар в корзину) и сервер определяет может он выполнить это действие или нет.

## Отсутствие состояния(Stateless)

Сервер не запоминает состояние клиента, он не помнит предыдущие запросы [[#^c73853|клиента]]. 

Каждый раз клиент и сервер общаются как в первый раз и каждый запрос на сервер должен содержать <u style='color:#7dcfff'>всю необходимую информацию</u> для его выполнения. ^8956ca

## Кэшировани(Caching)

Кэширование необходимо чтобы убрать нагрузку с сервера. Мы сохраняем часть данных, которые мы получили с сервера, чтобы заново не отправлять запросы и не нагружать сервер. Эти данные хранятся в браузере или приложении.

> [!example] Пример
>  Мы получили с сервера картинку и, чтобы заново не отправлять запросы на её получение, сервер говорит браузеру сохранить её у себя в кеше.

## Единство/единообразие интерфейса(Uniform Interface)

Клиент и сервер имеют общий и понятный язык для общение между собой. 
1. [[#^eb89c0|Всё к чему мы хотим обратиться на сервер имеет свой уникальный URL адрес]]
2. [[#^8956ca|При отправке запроса, запрос содерижт всю необходимую информацию о том что нужно сделать и какие данные нас интересуют.]] 
   > [!example]- Создание поста
>  Когда мы хотим создать пост в блоге, мы отправляем все данные о том, каким хочет видеть результат. Если редактируем пост, то снова отправляем все данные о том, каким хотим видеть результат, даже если часть из них не менялась. Запрос, по типу, просто изменить заголовок не пойдёт. Нужно полностью отправить данные о посте с изменённым заголовком, даже если всё остальное не менялось.
3. Самодостаточный сообщения. В ответе от сервера есть всё, чтобы клиент понял, как его обрабатывать.
   > [!example]- Поиск фото
> Сервер на запрос фотки группы Tokio Hotel. В ответ скажет что это изображение и клиент поймёт что с этим делать.
4. Клиенты могут получать информацию о других связанных ресурсах при помощи гипперсылок в ответ от сервера.

## API - Application Programming Interface

API - это интерфейс, который описывает как общаться с программой. Некое описание как программы общаются друг с другом - что можно спросить, что получить в ответ, что и куда для этого присылать. Endpoint = Метод.
