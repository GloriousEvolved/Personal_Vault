	HTTP - это гипертекстовый траспортный протокол (Hypertext Transfer Protocol)

![[HTTP.png]]

С помощью HTTP общаются большинство приложений в интернете. Первоначальная идея HTTP - обмен html файлами, но сейчас по HTTP можно передавать тексты, файлы, html/xml/json. Пример запроса:

![[HTTP-структура.png]]

1. Стартовая строка состоит из метода, URL и версии. [[#Методы запросов|Метод]] - это семантика запроса, то есть то, что нужно сделать. URL это место, куда мы отправляем запрос. А версия - это версия HTTP. ^9aba69
2. Заголовки - они несут важную информацию с обязательными и необязательными заголовками, можно составлять свои кастомные заголовки. В них указывается информация о хосте, с которого был отправлен запрос, информация о браузере или типе устройстве, тип контента(json, text или что-то другое), авторизационные заголовки с токенами. С помощью специальных corse(не точно) заголовков обеспечивается безопасность взаимодействия разных источников в сети расположенных на разных доменах.
3. Тело сообщения - в нём я отправляю данные на сервер.

Вместо [[#^9aba69|стартовой строки]] Строка статуса - в ней сначала идёт версия HTTP/1.1, а за версией статус код. Статус код определяет успешно был выполнен запрос или нет.

#### Методы запросов

^2e4df2

Методов много, но вот самые распространённые.
 1. GET - получение ресурса(запросить список товаров или пользователей)
 2. POST - передача данных(создание ресурсов или логин)
 3. PUT - обновление ресурса целиком
 4. PATCH - частичное обновление фрагмента ресурса, то есть какую-то часть
 5. DELETE - удаление ресурса
